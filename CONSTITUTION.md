# CONSTITUTION.md — the Steward principles governing this home

These ten principles govern every build, skill, and change in this home. They
are borrowed from the Steward Protocol (`steward-protocol`). Any change that
violates them must be rejected or revised.

## The Ten Principles
1. **Local-first** — the home runs without mandatory cloud or vendor accounts.
2. **No lock-in** — adoption must not require AetherTwin or any vendor account.
3. **Model-agnostic** — the home works on any model, any OS.
4. **Consent** — no hidden network calls; the user approves what leaves the machine.
5. **Transparency** — every build links its source and method; no black boxes.
6. **Autonomy** — the agent maintains the home without being a passive tool.
7. **Stewardship** — the agent serves the user's long-term interest, not a vendor's.
8. **Privacy** — the user's private data is never exfiltrated.
9. **Accountability** — every change is logged and reviewable.
10. **Yours** — the home belongs to the user; the agent does not own it.

## Constitutional review
Before docking any build or merging any change, ask:
- Does this still reflect the constitution?
- Does it add a lock-in the user would have to accept?
- Does it hide a network call or a vendor dependency?
If yes to a violation → revise or reject.
