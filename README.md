# Hermes Mothership

**This is the agent's home — not a website.**

It is the structured repo that persists Hermes (the agent) between sessions:
its charter, its identity, its skill registry, and the builds it docks. It can
be rendered or browsed, but the *repo is the home*. The home belongs to the
user (Amara) and is governed by the Steward constitution — local-first,
model/OS-agnostic, no vendor lock-in.

## What lives here

| Path | What it is |
|---|---|
| `AGENTS.md` | The home's operating charter — how the agent behaves here |
| `SOUL.md` | The agent's identity at home |
| `CONSTITUTION.md` | The Steward 10 Principles that govern this home |
| `skills/` | Registry of reusable procedures the agent maintains |
| `builds/` | Artifacts the agent builds and docks (e.g. `steward-card/`) |

## Principles

- **Local-first.** Nothing here requires a vendor account or cloud to exist.
- **No lock-in.** The home must run on any model / any OS. Adoption must not
  require AetherTwin or any vendor account.
- **Yours.** The user owns this home. The agent maintains it; it does not own it.
- **Transparent.** Every build links to its source and method. No black boxes.

## North star

`steward-protocol` — an open protocol for collaborative intelligence.
Adoption must not require AetherTwin.
